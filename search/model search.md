# 🛡️ [Phase 1.5] 악성코드 탐지 아키텍처 결정을 위한 다중 모델 심층 교차 분석 (Model Selection & Cross-validation Analysis)

본 문서는 정적 API 행위 데이터를 기반으로 악성코드를 탐지하기 위한 최적의 기계학습 아키텍처를 선정하기 위해 작성되었습니다. 선형 모델부터 심층 신경망까지 5가지 주요 알고리즘을 교차 검증(Cross-validation)하고, 본 프로젝트의 데이터 특성(희소 이진 벡터, Test Set 1:23 불균형, 4만 개 샘플)과 보안 도메인의 요구사항(실시간 탐지, 설명 가능성)을 기준으로 타당성을 평가합니다.

---

## 1. 로지스틱 회귀 (Logistic Regression, LR)
가장 기초적인 선형 분류 모델로, 본 프로젝트의 성능 기준선(Baseline)을 설정하기 위해 평가되었습니다.

* **기저 논리**: 추출된 30개의 핵심 API 변수에 각각 가중치(Weight)를 곱하고 합산한 뒤, 시그모이드(Sigmoid) 함수를 통해 악성 확률(0~1)을 도출합니다.
* **장점 (Pros)**: 
  * 연산량이 극도로 적어 추론 속도가 압도적으로 빠릅니다 (1ms 미만).
  * 각 API의 가중치를 통해 탐지 근거를 직관적으로 해석할 수 있습니다.
* **단점 (Cons)**: 
  * 악성코드 특유의 **비선형적 상호작용**을 전혀 파악하지 못합니다. 
  * **불균형 데이터에 대한 극명한 취약성**: Test Set의 1:23(정상:악성) 극심한 불균형 환경에서, 모든 것을 '악성'으로 찍어도 Accuracy가 96%가 나오는 정확도의 역설(Accuracy Paradox)에 빠집니다. 실제로 LR은 정상 파일을 악성으로 오판하는 비율(False Positive)이 매우 높게 측정되었습니다.
* **학술적 근거**: 
  > *Ucci, D., et al. (2019). Survey of machine learning techniques for malware analysis.*
  > (선형 모델은 API 간의 복잡한 행위 패턴을 포착하는 데 구조적 약점을 지니며, 고도화된 탐지에는 부적합함이 입증됨)

## 2. 서포트 벡터 머신 (Support Vector Machine, SVM)
정상 파일과 악성 파일을 가르는 최적의 기하학적 초평면(Hyperplane)을 찾는 공간 분할 모델입니다.

* **기저 논리**: 데이터 포인트(Support Vector)를 기준으로 클래스 간의 거리(Margin)를 극대화하여 구조적 위험을 최소화합니다.
* **장점 (Pros)**: 
  * 0과 1로 이루어진 희소 행렬(Sparse Matrix) 공간에서 수학적으로 매우 견고합니다.
* **단점 (Cons)**: 
  * 4만 개 수준의 데이터셋에서 학습 알고리즘의 시간 복잡도가 $O(n^2)$ 이상으로 기하급수적으로 증가합니다. 
  * 극심한 클래스 불균형(1:23)을 해소하기 위해 커널(RBF 등)과 규제 파라미터(C, Gamma)를 조정하는 과정이 매우 불안정하고 과적합을 유발하기 쉽습니다.
* **학술적 근거**: 
  > *Kolter, J. Z., & Maloof, M. A. (2006). Learning to detect and classify malicious executables in the wild.*
  > (정적 특성 분석에서 대규모 데이터셋으로의 확장성(Scalability) 한계와 실시간 튜닝의 어려움을 지적)

## 3. K-최근접 이웃 (K-Nearest Neighbors, K-NN)
수학적 공식이나 트리 분기 없이, 오직 데이터 간의 '유사도(거리)'만을 측정하는 인스턴스 기반 모델입니다.

* **기저 논리**: 미상의 `.exe` 파일이 유입되면, 훈련 데이터 중 API 패턴이 가장 유사한 K개의 파일을 찾아내어 다수결로 판별합니다.
* **장점 (Pros)**: 사전 학습(Training) 단계가 없는 게으른 학습(Lazy Learning) 방식으로, 논리가 투명합니다.
* **단점 (Cons)**: 
  * **실시간 런타임 탐지 환경에 치명적**입니다. 훈련은 0초지만, 파일 하나를 검사할 때마다 4만 개의 훈련 샘플 전체와 유클리디안/맨해튼 거리를 일일이 계산해야 하므로 시스템 오버헤드가 극심합니다.
* **학술적 근거**: 
  > *Firdausi, I., et al. (2010). Analysis of machine learning techniques for malware detection.*
  > (K-NN은 사전 학습이 없으나 추론 단계의 막대한 연산량으로 인해 엔드포인트 보안(EDR) 환경에 부적합함)

## 4. XGBoost (eXtreme Gradient Boosting)
트리 기반 앙상블 모델 중 부스팅(Boosting)의 대표 주자로, 정형 데이터 분류의 SOTA(State-of-the-Art) 모델입니다.

* **기저 논리**: 이전 트리가 잘못 예측한 오차(Residual)를 다음 트리가 순차적으로 보완해 나가는 경사 하강법(Gradient Descent)을 적용합니다.
* **장점 (Pros)**: 예측 정확도 측면에서 한계치까지 성능을 끌어올릴 수 있으며, 미세한 비선형적 경계를 정밀하게 분리합니다.
* **단점 (Cons)**: 
  * 순차 보정의 특성상 악성코드 데이터셋에 흔히 존재하는 '노이즈(Noise)'까지 과도하게 학습해버리는 **과적합(Overfitting) 위험**이 매우 높습니다. 
  * Train 셋(SMOTE 적용 1:1)과 Test 셋(1:23 분포)의 분포 불일치 상황에서 모델 편향(Bias)이 급격히 무너질 우려가 있습니다.
* **학술적 근거**: 
  > *Chen, T., & Guestrin, C. (2016). XGBoost: A scalable tree boosting system.*
  > (오차를 순차적으로 줄이는 부스팅 특성상 노이즈가 많은 보안 도메인에서는 과적합 방지를 위한 극도의 튜닝이 요구됨)

## 5. 심층 신경망 (Deep Neural Network, DNN / MLP)
여러 겹의 은닉층(Hidden Layer)을 통해 데이터의 계층적 특징을 스스로 학습하는 딥러닝 방법론입니다.

* **기저 논리**: 30차원의 API 이진 벡터를 비선형 활성화 함수(ReLU 등)를 통과시키며 고차원적인 잠재 특성(Latent Feature)을 조합합니다.
* **장점 (Pros)**: 사람이 직관적으로 파악할 수 없는 수십 개 API 간의 심층적인 상호작용을 포착합니다.
* **단점 (Cons)**: 
  * **블랙박스(Black-box) 한계**: "이 파일을 왜 악성코드로 판단했는가?"에 대한 해석력(Explainability)이 전무합니다. 탐지 근거를 명확히 남겨야 하는 보안 파이프라인에서 치명적입니다.
  * 30차원으로 압축된 이진(Binary) 정형 데이터에서는 신경망의 이점이 크지 않으며 트리 기반 모델이 구조적으로 우세합니다.
* **학술적 근거**: 
  > *Saxe, J., & Berlin, K. (2015). Deep neural network based malware detection using two dimensional binary program features.*
  > (DNN의 성능을 입증함과 동시에, 보안 분석가가 판별 근거를 역추적하기 어려운 해석 가능성의 구조적 한계를 명시)

# 🛡️ 악성코드 탐지 아키텍처: 랜덤 포레스트(Random Forest) 선정에 대한 학술적 근거

본 문서는 정적 API 행위 데이터를 기반으로 악성코드를 탐지하기 위한 주력 기계학습 모델로 **랜덤 포레스트(Random Forest)**를 선정한 학술적, 논리적 타당성을 증명하기 위해 작성되었습니다. 

단순히 '정확도가 높아서'가 아닌, 우리 프로젝트의 특수한 보안 데이터 환경(희소 이진 벡터, 극심한 클래스 불균형, 실시간 탐지 요구사항)에서 랜덤 포레스트가 가지는 독보적인 이점을 전문 학술지와 논문을 근거로 서술합니다.

---

## 1. 악성코드 행위의 '비선형적 상호작용(Non-linear Interaction)' 포착

악성코드는 단일 API(예: `VirtualAlloc`) 호출만으로 악성 여부가 결정되지 않습니다. 특정 API들이 **"어떤 순서와 조합으로 함께 사용되었는가?"**라는 복합적이고 비선형적인 조건이 탐지의 핵심입니다.

* **타 모델의 구조적 약점**: 로지스틱 회귀(Logistic Regression)나 SVM(선형 커널) 등은 각 변수에 가중치를 단순 합산하므로, API 간의 복잡한 조건부 패턴("A API와 B API가 동시에 쓰일 때만 악성")을 학습하는 데 한계가 있습니다.
* **랜덤 포레스트의 우위**: 다수의 의사결정나무가 조건부 분기(If-Then-Else)를 연속적으로 형성하여, 단순한 선형 가중치로는 잡아낼 수 없는 상위 30개 핵심 API 간의 비선형적 상호작용을 완벽하게 공간 분할하여 학습합니다.

> **📚 참고 문헌 (Reference)**
> *Ucci, D., Aniello, L., & Baldoni, R. (2019). Survey of machine learning techniques for malware analysis. Computers & Security, 81, 123-147.*
> (본 논문에서는 악성코드 분석에 있어 선형 모델의 한계를 지적하며, 비선형 트리 앙상블 모델이 API 기반 분석에서 뛰어난 성능을 보임을 입증하였습니다.)

## 2. 배깅(Bagging)을 통한 과적합 방지와 노이즈 저항성

보안 데이터셋, 특히 윈도우 API 데이터는 정상 파일임에도 특이한 API를 호출하는 '노이즈(Noise)'가 섞여 있거나 변종 악성코드가 존재할 확률이 매우 높습니다.

* **타 모델의 구조적 약점**: 부스팅 계열(XGBoost, LightGBM)이나 단일 의사결정나무는 이전의 오차를 집요하게 줄여나가기 때문에, 훈련 데이터에 섞인 노이즈까지 과도하게 암기해버리는 **과적합(Overfitting)** 위험이 높습니다. 특히 Train 데이터와 Test 데이터의 분포가 극심하게 다를 때(1:23 불균형 등) 치명적입니다.
* **랜덤 포레스트의 우위**: 무작위 복원 추출(Bootstrap)을 통해 서로 다른 수백 개의 트리를 독립적으로 학습시키고, 그 결과를 **다수결(Voting)**로 통합합니다. 특정 트리가 노이즈에 의해 잘못된 판단을 내리더라도, 나머지 다수의 정상적인 트리가 이를 보정하여 '일반화(Generalization)' 성능을 극대화합니다.

> **📚 참고 문헌 (Reference)**
> *Breiman, L. (2001). Random forests. Machine learning, 45(1), 5-32.*
> (랜덤 포레스트의 창시자인 Breiman은 이 논문에서 트리들이 독립적으로 생성될 때, 강한 대수의 법칙(Strong Law of Large Numbers)에 의해 숲(Forest)이 커져도 과적합되지 않음을 수학적으로 증명했습니다.)

## 3. 사이버 보안 도메인의 필수 요건: '설명 가능성(Explainability)'

사이버 보안 시스템에서는 단순히 "이 파일은 90% 확률로 악성코드입니다"라는 결과뿐만 아니라, **"왜 악성코드로 판단했는가?"**에 대한 명확한 근거(근원지 추적)가 필요합니다.

* **타 모델의 구조적 약점**: 심층 신경망(DNN)과 같은 딥러닝 모델은 성능은 우수하나, 내부 구조가 '블랙박스(Black-box)'로 되어 있어 어느 API가 판별에 결정적이었는지 보안 분석가가 역추적하기 어렵습니다.
* **랜덤 포레스트의 우위**: 트리 분기 과정에서 불확실성(지니 불순도, Gini Impurity)을 가장 크게 떨어뜨린 변수들의 기여도를 수치화하여 **특징 중요도(Feature Importance)**를 직관적으로 제공합니다. (실제로 Phase 1 전처리 단계에서 상위 30개 API를 추출할 때 이 지표를 핵심 근거로 활용했습니다.)

## 4. 데이터 낭비 없는 OOB(Out-of-Bag) 자체 성능 검증

고품질의 악성코드/정상 파일 데이터는 수집과 라벨링에 많은 비용이 듭니다.

* **타 모델의 구조적 약점**: 일반적인 기계학습 모델은 평가를 위해 데이터를 훈련용(Train)과 검증용(Validation)으로 강제 분할해야 하므로, 훈련에 투입되는 절대적인 데이터 양이 10~20%가량 손실됩니다.
* **랜덤 포레스트의 우위**: 배깅 과정에서 통계적으로 전체 데이터의 약 **36.8%**는 추출되지 않고 잉여 데이터(OOB 샘플)로 남게 됩니다. 랜덤 포레스트는 이 36.8%의 데이터를 별도의 검증 세트로 낭비하지 않고 모델의 자체 모의고사용으로 즉시 활용하여, 데이터 손실 없이 완벽한 편향-분산 트레이드오프(Bias-Variance Trade-off) 평가를 수행합니다.

---

### 🏆 소결 (Conclusion)

본 프로젝트는 1:23이라는 극심한 Test Set 불균형 환경과 밀리초(ms) 단위의 실시간 탐지 환경을 전제로 합니다. 

랜덤 포레스트는 K-NN이나 SVM처럼 실시간 추론 시 계산 병목($O(n^2)$ 이상)이 발생하지 않으며, XGBoost처럼 과적합의 함정에 빠지지 않고 굳건한 재현율(Recall)을 유지합니다. 이에 더해 보안 시스템에 필수적인 **설명 가능성(해석력)**을 보장하므로, 본 악성코드 탐지 아키텍처의 최종 분류 모델로서 가장 학술적이고 실무적인 타당성을 지닙니다.
---
## 🏆 결론: 최종 아키텍처 채택 (Random Forest)

위 5가지 대안 모델들과의 교차 분석 결과, 본 프로젝트의 최종 탐지 아키텍처로 **랜덤 포레스트(Random Forest)**를 확정합니다.

1. **비선형 상호작용 및 차원의 저주 해결**: LR의 한계를 넘어 30개 핵심 API 간의 복합적 조건을 트리 분기로 완벽히 잡아냅니다.
2. **실시간 탐지의 현실성**: SVM, KNN과 달리 훈련 후 추론(Inference) 속도가 밀리초(ms) 단위로 처리되어 실시간 방어망(Phase C) 구축이 가능합니다.
3. **불균형 데이터 및 과적합에 대한 극강의 방어력 (핵심 근거)**: XGBoost의 과적합 위험이나 DNN의 블랙박스 문제를 완벽히 덮어냅니다. 다수의 독립된 트리를 통합하는 **배깅(Bagging)** 메커니즘은 노이즈에 강건하며, SMOTE로 균형을 맞춘 Train 데이터와 1:23으로 극심하게 불균형한 Test 데이터의 분포 차이에서도 재현율(Recall) 하락을 방어할 수 있는 가장 안정적인 통계적 근거를 제공합니다.
