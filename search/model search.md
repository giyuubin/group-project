# 🛡️ [Phase 1.5] 악성코드 탐지 아키텍처 결정을 위한 다중 모델 심층 분석 (Model Analysis)

## 1. 분석 개요
- **목적**: 정적 API 행위 데이터를 활용한 최적의 악성코드 탐지 모델 선정 및 학술적 타당성 검증
- **데이터 환경**:
    - 약 4만 개의 샘플 및 1,000개 API의 희소 이진 벡터(Sparse Binary Vector)
    - 특징 선택(Feature Selection)을 통한 상위 30개 핵심 API 피처 추출 완료
    - Test Set 기준 1:23(정상:악성)의 극심한 클래스 불균형 환경
- **핵심 요구사항**: 실시간 탐지 성능, 모델의 설명 가능성(Interpretability), 과적합(Overfitting) 방지

---

## 2. 모델별 심층 비교 분석 및 타당성 검토

### 2.1. 로지스틱 회귀 (Logistic Regression, LR)
- **기저 논리**: API 변수별 가중치 합산 후 시그모이드 함수를 통해 악성 확률을 도출하는 선형 모델
- **장점**: 단일 샘플에 대한 추론 속도가 매우 빠르며(1ms 미만), 가중치를 통한 직관적 해석이 가능함
- **단점**:
    - API 간의 복잡한 비선형적 상호작용(예: 특정 API들의 동시 호출 패턴) 포착 불가
    - **정확도의 역설(Accuracy Paradox)**: 1:23 불균형 환경에서 압도적 다수인 '악성'으로 편향 예측함. 이로 인해 정확도는 높게 나타나나, 소수 클래스인 '정상 파일'을 악성으로 오판하는 False Positive Rate가 극단적으로 높음
- **학술적 근거**: *Ucci et al. (2019)*에 따르면 선형 모델은 지능화된 맬웨어의 복잡한 행위 패턴을 모델링하기에 구조적 한계가 있음

### 2.2. 서포트 벡터 머신 (Support Vector Machine, SVM)
- **기저 논리**: 클래스 간 마진(Margin)을 극대화하는 최적의 초평면을 찾는 공간 분할 모델
- **장점**: 고차원 희소 벡터 공간에서 수학적으로 견고한 분리 성능을 제공함
- **단점**:
    - 데이터 샘플(4만 개) 증가 시 시간 복잡도가 기하급수적으로 상승하여 대규모 학습 시 병목 발생
    - 1:23 불균형 환경에서 하이퍼파라미터(C, Gamma) 튜닝이 매우 불안정하며 과적합 위험이 존재함
- **학술적 근거**: *Kolter & Maloof (2006)*는 정적 분석 내 SVM의 우수성을 언급했으나 실시간 유지보수성과 확장성 결여를 지적함

### 2.3. K-최근접 이웃 (K-Nearest Neighbors, K-NN)
- **기저 논리**: 유입된 샘플과 기존 데이터 간의 유사도(거리)를 측정하여 다수결 판별
- **장점**: 별도의 학습 단계가 필요 없는 직관적 모델 구조를 가짐
- **단점**: **실시간 보안 파이프라인에 부적합함**. 추론 시마다 4만 개 전체 데이터와 거리를 일일이 계산해야 하므로 시스템 오버헤드가 극심함
- **학술적 근거**: *Firdausi et al. (2010)*의 연구 결과, K-NN은 추론 단계의 막대한 연산량으로 인해 실시간 엔드포인트 보안 환경에 부적합함이 증명됨

### 2.4. XGBoost (Gradient Boosting)
- **기저 논리**: 이전 트리의 잔차(Residual)를 순차적으로 보정하며 학습하는 부스팅 모델
- **장점**: 예측 정확도가 매우 높으며 비선형 결정 경계를 정밀하게 탐색함
- **단점**:
    - 보안 데이터에 포함된 사소한 노이즈까지 과도하게 암기하는 과적합(Overfitting) 성향이 강함
    - 1:23 불균형 테스트셋 환경에서 모델 편향이 무너질 우려가 커 정교한 튜닝이 필수적임
- **학술적 근거**: *Chen & Guestrin (2016)*은 부스팅 계열 모델이 노이즈가 많은 환경에서 성능 하락 방지를 위한 복잡한 제어가 필요함을 명시함

### 2.5. 심층 신경망 (DNN / MLP)
- **기저 논리**: 다층 은닉층을 통해 API 간의 계층적 잠재 특징을 추출하는 딥러닝 모델
- **장점**: 고차원 API 상호작용을 스스로 학습하여 새로운 변종에 대한 대응 잠재력을 지님
- **단점**:
    - **블랙박스(Black-box) 문제**: 탐지 결과에 대한 구체적 근거 제시가 어려워 보안 신뢰성 확보에 제약이 있음
    - 본 프로젝트와 같은 이진 정형 데이터(Tabular Data) 환경에서는 트리 기반 모델 대비 성능적 우위가 낮음
- **학술적 근거**: *Saxe & Berlin (2015)*은 DNN의 우수성과 동시에 보안 분석가의 해석 가능성 부재를 핵심 한계로 지적함

## 3. 최종 아키텍처 확정: 랜덤 포레스트 (Random Forest)

다양한 모델들의 교차 검증 결과, 본 프로젝트의 최종 탐지 모델로 **랜덤 포레스트**를 채택함. 채택 근거는 다음과 같음

1. **비선형 상호작용 포착**: 조건부 분기 구조를 통해 30개 핵심 API 간의 복잡한 호출 패턴을 효과적으로 학습함
2. **배깅(Bagging) 기반 강건성**: 독립적인 다수결 시스템을 통해 노이즈와 변종 악성코드에 대한 과적합을 방어함
3. **설명 가능성(Explainability)**: 특징 중요도(Feature Importance) 수치를 제공하여 보안 분석가에게 명확한 차단 근거를 제시함
4. **OOB(Out-of-Bag) 검증**: 36.8%의 미학습 데이터를 활용해 데이터 낭비 없이 실시간으로 모델 성능을 자체 검증함
5. **실시간성 최적화**: 추론 속도가 밀리초(ms) 단위로 안정적이며 대규모 확장 환경에 적합함
