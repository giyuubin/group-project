1. 프로젝트 개요 // 용어 확실하게 알고있기 나부터..
   - 실행 파일(.exe)의 PE 구조에서 API 호출 패턴을 직접 추출하고, 이를 학습한 자체 기계학습 모델을 통해
     악성 여부를 사전에 판별하는 파이프라인 구축
   - 기존 백신과의 차별점:
     - 서명 기반(Signature-based) 탐지는 과거 수집된 해시 데이터에 의존하므로 제로데이 취약점이나
       매번 코드가 변형되는 다형성 악성코드에 구조적인 취약성을 보인다
     - 동적 분석(Dynamic Analysis)는 샌드박스를 이용해 직접 파일을 실행해보는 방식으로, 정밀하지만
       분석 환경을 우회하는 맬웨어가 존재하며 과도한 시스템 리소스를 소모한다

3. [예정] 알고리즘 튜닝 및 **OOB 기반 자체 성능 검증** // 우리 이거 이용합시다! 추가로 교수님 강의 교안 이용해도 좋을 것 같아요!
   - 데이터 낭비 없는 OOB(Out-Of-Bag) 36.8% 자체 검증 도입:
     - 일반적인 기계학습은 평가를 위해 데이터 훈련용과 검증용으로 분리하므로 데이터 낭비가 발생함
     - 우리는 앙상블 기법인 랜덤 포레스트의 복원 추출(Bagging) 과정에서 통계적으로 전체 데이터의 약 36.8%가 트리를 학습할 때 단 한 번도 뽑히지 않고 남게 되는 특성을 활용함
     - 이 미학습된 OOB 샘플들을 모델의 자체 검증 데이터로 활용하여, 별도 데이터 낭비 없이 훈련과 동시에 평가 지표 도출하고 과적합 방어 가능
   - **970개의 데이터가 걸러졌는데 OOB가 의미가 있을까?**
     - 여기서 걸러진 970개는 1000개의 질문 중 불필요한 질문(Feature, 열)을 버린 것이며, 학습시 풀어야 할 4만 개의 소프트웨어 파일(Sample, 행) 데이터의 양은 그대로 유지됨
     - 따라서 OOB 검증은 "압축해 낸 30개의 핵심 질문(API)이 훈련 중 한 번도 보지 못한 36.8%의 새로운 파일(OOB 샘플)들에게도 통용되는가?"를 증명해준다
